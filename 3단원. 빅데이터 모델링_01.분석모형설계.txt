CHAPTER01. 분석 모형 설계

	1. 분석 절차 수립
	
		a. 분석 모형 선정
		
현상에서 패턴을 발견하는 것 - 탐색적 데이터 분석 (EDA)
인과적인 결론을 도출하는 것 - 통계적 추론
예측하는 것 - 기계학습(머신러닝)

(1) 통계기반 분석 모형 선정
	- 기술 통계 (Descriptive Statistics)
	- 상관 분석
	- 회귀 분석
	- 분산 분석
	- 주성분 분석
	- 판별 분석

(2) 데이터 마이닝 기반 분석 모형 선정
	- 데이터 마이닝 개념 : 대용량 데이터로 부터 패턴, 관계 혹은 규칙 등을 탐색하고 통계적인 기법을 활용하여 모델화
	
	- 데이터 마이닝 기바 분석 모델 분류
	분류모델(범주형, 이산형 변수의 범주를 예측) : 통계적 기법 = 로지스틱 회귀분석과 판별분석, 트리 기반 기법, 최적화 기법 = 서포트 벡터 머신, 기계학습
	
	예측모델(범주형, 수치형 등의 과거 데이터로 부터 다른 데이터의 결과값 예측) : 회귀분석, 의사결정나무, 시계열 분석, 인공신경망
	
	군집화 모델 : 계층적 방법 ( 응집 분석법/ 분할 분석법 ), 비계층적 방법 (k-means)
	
	연관규칙 모델 = 장바구니 분석 (규칙을 수치화 하는 것)
	
	
(3) 머신러닝 기반 분석 모형 선정
	- 지도학습 : 정답 레이블(Label)이 포함되어 있는 학습 데이터/ 설명변수와 목적변수 간의 관계성을 표현하거나 미래를 예측
	Ex) 로지스틱 회귀분석, 인공신경망 분석, 의사결정나무, 서포트 벡터 머신, 랜덤 포레스트, 감성 분석
	- 비지도학습 = 자율학습 : 정답 레이블이 없는 상태/ 주로 현상의 설명이나 특징 도출, 패턴 도출 등의 문제에 많이 활용/ 데이터 마이닝 성격
	Ex) 군집화, 차원축소 기법, 연관 관계분석, 자율학습 인공신경망(자기 조직화 지도), 딥러닝
	
	- 강화학습
	- 준지도학습
	

(4) 변수에 따른 분석기법 선정 -> 변수의 유형 및 개수를 확인
	- 변수의 개수에 따른 분석기법
	단일변수 분석
	이변수 분석
		- 연속형 x 연속형 = 산점도/ 상관분석
		- 범주형 x 범주형 = 누적 막대그래프/ 카이제곱 분석 - 두변수가 독립적인지
		- 범주형 x 연속형 = 누적 막대그래프/ T- test (2개), ANOVA (3개) - 집단 별 평균차가 유의한지
	다변수 분석
	- 독립변수와 종속변수의 데이터 유형에 따른 분석기법

		종속변수

		인공신경망 모델	판별분석
		K-최근접 이웃기법	K-최근접 이웃기법

(5) 분석기법 선정 고려사항

(6) 분석 모형 활용 사례
	- 연관규칙학습 : 변인 간 상관관계 ex) 커피를 구매하는 사람이 탄산음료 많이 구매?
	- 분류 분석 : 문서, 조직을 나눌 때 ex) 이 사용자는 어떤 특성을 가진 집단에 속하는가?
	- 유전자 알고리즘 : 자연선택, 돌연변이 ex) 응급실에서 응급처치 프로세스
	- 기계학습 : 알려진 특성 활용하여 훈련 데이터 학습시키고 예측 ex) 기존 시청기록 바탕으로 영화추천
	- 회귀분석 : 독립변수와 종속변수 관계파악 ex) 구매자 나이가 구매차량의 유형에 미치는 영향
	- 감성분석 : ex) 환불 정책에 대한 고객의 평가
	- 소셜 네트워크 분석 : 사람과의 관계 파악 ex) 고객들 간의 관계망 구성




		B. 분석 모형 정의
(1) 분석 모형 정의 개념 : 분석 모형을 선정하고 모형에 적합한 변수를 선택하여 모델의 사양을 작성
	- 파라미터 : 모델 내부에서 확인이 가능한 변수
	Ex) 인공신경망의 가중치, 서포트 벡터 머신에서의 서포트 벡터, 선형회귀 또는 로지스틱 회귀 분석에서의 결정계수
	- 하이퍼 파라미터 : 모델 외적인 요소로 사용자가 지정해주는 값 (경험에 의해서 결정)
	Ex) 신경망에서의 학습률, 의사결정나무에서의 나무의 깊이, knn 에서의 k의 개수

(2) 분석 모형 정의 고려사항
	- 과대적합, 과소적합 주의
	- 부적합 모형 현상 : 모형 선택 오류, 변수 누락, 부적합 변수 생성, 동시 편향


		C. 분석 모형 구축 절차
(1) 요건 정의
분석요건 도출 -> 수행방안 설계 -> 요건 확정 단계

(2) 모델링
모델링 마트 설계 및 구축 -> 탐색적 분석과 유의 변수 도출 -> 모델링 -> 모델링 성능 평가

(3) 검증 및 테스트
운영 상황에서 실제 테스트 -> 비즈니스 영향도 평가

(4) 적용
운영 시스템에 적용과 자동화 -> 주기적 리모델링

		B. 데이터 분할
(1) 데이터 분할 개념 : 과대적합 문제를 예방하여 2종 오류인 잘못된 귀무가설을 채택하는 오류를 방지하는데 목적이 있다.
(2) 데이터 분할 시 고려사항 : 학습용, 검증용(60-80%), 평가용 데이터(20-40%)
-> 데이터가 충분하지 않을 때는 검증용을 제외
